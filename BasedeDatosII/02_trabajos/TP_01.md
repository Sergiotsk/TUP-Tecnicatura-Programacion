## Repaso de SQL

### 1. Modelo Relacional

En función del siguiente Modelo Relacional dar de alta las tablas correspondientes.

#### Tabla: Empleado
| Campo | Tipo |
|---|---|
| ID (PK) | NUMBER(4) |
| Apellido | VARCHAR2(25 CHAR) |
| Nombre | VARCHAR2(25 CHAR) |
| Alta | DATE |
| Departamento_ID (FK) | NUMBER(4) |
| Dirección | VARCHAR2(40 CHAR) |
| Telefono | VARCHAR2(12) |

#### Tabla: Departamento
| Campo | Tipo |
|---|---|
| ID (PK) | NUMBER(4) |
| Nombre | VARCHAR2(30 CHAR) |

#### Tabla: Proyecto
| Campo | Tipo |
|---|---|
| ID (PK) | NUMBER(4) |
| Nombre | VARCHAR2(40 CHAR) |
| FInicio | DATE |
| FFin | DATE |

#### Tabla: pertenece *(relación Empleado ↔ Proyecto)*
| Campo | Tipo |
|---|---|
| Empleado_ID (PFK) | NUMBER(4) |
| Proyecto_ID (PFK) | NUMBER(4) |

### a. Insertar registros

En cada una de las tablas creadas insertar la cantidad de registros que se indican a continuación:

- Empleados → 7 registros
- Departamentos → 3 registros
- Proyectos → 2 registros
- Pertenece → 4 registros *(relación entre proyectos y empleados)*

### b. Modificación de tabla

Agregar en la tabla **Proyecto** el campo cantidad de días estimados de duración del proyecto.

### c. Consultas

- Listado de Empleados por proyecto *(nombre del empleado y nombre del proyecto)*
- Cantidad de Empleados por Departamento
- Fecha Posible de culminación de cada Proyectos
- Empleado con mayor antigüedad por Departamento
